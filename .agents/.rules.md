# RailUndo — Design System & UI/UX Rules
> **This file is the single source of truth for all UI/UX decisions.**
> Every component, page, and prompt must follow these rules without exception.
> If a rule conflicts with a library default, override the library.
 
---
 
## 1. Philosophy
 
RailUndo is a **utility-first, zero-friction mobile tool**. Users open it while sitting in traffic behind a closed railway gate. They are stressed, impatient, and on a 4G connection. Every design decision must serve one goal: **get the user the answer in under 3 seconds, with one thumb**.
 
Design references studied:
- `currentundo.com` — dark theme, PIN-first, no login, live feed (50K users in 2 weeks)
- `kuzhiyundo.com` — tap-to-report pattern, mobile-first list layout
- Kerala Undo Universe conventions — Malayalam-friendly, low-data, PWA
---
 
## 2. Color Palette
 
### Theme: Dark by default (not toggleable in v0)
Kerala users are often outdoors in bright sunlight or inside vehicles — dark theme reduces glare and saves battery on OLED phones.
 
```
/* Background layers */
--bg-base:        #0A0A0A   /* Page background */
--bg-surface:     #141414   /* Cards, list items */
--bg-elevated:    #1E1E1E   /* Modals, bottom sheets */
--bg-input:       #242424   /* Input fields */
 
/* Status colors — high contrast, colorblind-safe */
--status-open:    #22C55E   /* Green — gate is OPEN */
--status-closed:  #EF4444   /* Red   — gate is CLOSED */
--status-unknown: #6B7280   /* Gray  — no recent report */
 
/* Status backgrounds (subtle fills) */
--status-open-bg:    rgba(34, 197, 94, 0.12)
--status-closed-bg:  rgba(239, 68, 68, 0.12)
--status-unknown-bg: rgba(107, 114, 128, 0.10)
 
/* Text */
--text-primary:   #F5F5F5   /* Headings, gate names */
--text-secondary: #A3A3A3   /* Subtitles, metadata */
--text-muted:     #525252   /* Timestamps, hints */
 
/* Brand accent */
--accent:         #FACC15   /* Yellow — RailUndo brand, CTAs, active states */
--accent-dim:     rgba(250, 204, 21, 0.15)
 
/* Borders */
--border:         rgba(255, 255, 255, 0.08)
--border-strong:  rgba(255, 255, 255, 0.15)
 
/* Danger / Alert */
--danger:         #F97316   /* Orange — used for "gate closed long time" warning */
```
 
### Never use:
- White backgrounds in any component
- Pure `#000000` black (use `--bg-base` instead)
- Green/red text on dark green/red backgrounds (fails contrast check)
- More than 2 colors in a single component

## 5. Touch Targets & Accessibility
 
These are non-negotiable for a mobile utility app:
 
- **Minimum tap target: 44×44px** (Apple HIG standard). Every button, chip, and interactive element must meet this.
- Status report buttons ("Open" / "Closed") must be **minimum 52px tall** — users tap these in moving vehicles.
- Never place two tappable elements closer than **8px apart**.
- All interactive elements must have a **visible focus ring** for keyboard/accessibility use: `outline: 2px solid var(--accent); outline-offset: 2px`
- Color must never be the ONLY indicator of status — always pair color with a text label or icon.
- Images and icons must have `aria-label` or `alt` text.
- Use semantic HTML: `<button>` for actions, `<a>` for navigation, `<header>`, `<main>`, `<section>` for structure.
### WCAG AA compliance (required):
- Text on dark backgrounds must be ≥ 4.5:1 contrast ratio
- `--text-primary` (#F5F5F5) on `--bg-surface` (#141414) = **14.5:1** ✅
- `--status-open` (#22C55E) on `--bg-surface` (#141414) = **5.8:1** ✅
- `--status-closed` (#EF4444) on `--bg-surface` (#141414) = **5.1:1** ✅
- `--accent` (#FACC15) on `--bg-base` (#0A0A0A) = **13.2:1** ✅

## 6. Components
 
### 6.1 Gate Card
The most important component. Must be scannable in 1 second.
 
```
Layout:
[ Status dot ] [ Gate name          ] [ Status badge ]
               [ District · subtext ]
               [ ○ 4 min ago · 3 reports           ]
 
- Status dot: 10px circle, color matches status
- Gate name: 16px/600, truncate with ellipsis if > 28 chars
- Status badge: pill shape, 12px/700, colored bg + colored text
- Metadata row: 12px/400, --text-muted
- Right side: status badge right-aligned
- Entire card is tappable (opens detail / report sheet)
```
 
### 6.2 Status Badge
```
Pill: border-radius 999px, padding 4px 10px
OPEN    → bg: --status-open-bg,    text: --status-open,    "● OPEN"
CLOSED  → bg: --status-closed-bg,  text: --status-closed,  "● CLOSED"
UNKNOWN → bg: --status-unknown-bg, text: --status-unknown, "○ UNKNOWN"
```
 
### 6.3 District Filter Chips
```
Horizontal scroll row, no scrollbar visible (overflow-x: auto, scrollbar-width: none)

  - padding: 8px 16px

  - min-height: 36px (meets 44px tap target with margin)
  - Inactive: bg --bg-elevated, border --border, text --text-secondary
  - Active:   bg --accent-dim,  border --accent, text --accent
  - Font: 13px/600
Chips: "All", "Kannur", "Kozhikode", "Thrissur", "Ernakulam", "Palakkad", "Others"

### 6.4 Report Bottom Sheet
```
- Triggered by tapping a gate card
- Slides up from bottom, 60% screen height
- Background: --bg-elevated, border-radius 20px top corners only
- Backdrop: rgba(0,0,0,0.7) blur(4px)
- Handle bar: 4px × 36px, --border-strong, centered at top
- Two large buttons side by side:
    OPEN button:   bg --status-open,   text #000, icon: 🟢, 52px tall
    CLOSED button: bg --status-closed, text #fff, icon: 🔴, 52px tall
- Cancel: text button, --text-muted, below the two buttons
- Gate name shown at top of sheet: 16px/600
```
 
### 6.5 Header
```
- Height: 56px
- Background: --bg-base with bottom border: 1px solid --border
- Left: "🚂 railundo" — emoji + wordmark, 18px/700
- Right: live indicator dot (pulsing green if realtime connected)

### 6.6 Live Indicator
```

- Title: "No gates found" 16px/600 --text-primary
- Subtitle: "Try a different district" 14px --text-secondary
```

## 7. Motion & Animation
 
- **No decorative animations** — every animation must serve a functional purpose.
- Status change (e.g. gate flips from CLOSED → OPEN): **background flash** using `transition: background 0.3s ease`
- Bottom sheet open/close: `transform: translateY` with `transition: 0.25s ease-out`
- Live indicator pulse: CSS `@keyframes` — subtle opacity pulse, 2s loop
- Report success: brief **green flash** on the card + "Thanks for reporting!" toast
- Toast: slides in from bottom, auto-dismiss after 2.5s
- Respect `prefers-reduced-motion`: wrap all animations in `@media (prefers-reduced-motion: no-preference)`
```css
@keyframes pulse-live {
  0%, 100% { opacity: 1; transform: scale(1); }
  50%       { opacity: 0.5; transform: scale(0.85); }
}
 
@keyframes slide-up {
  from { transform: translateY(100%); }
  to   { transform: translateY(0); }
}
 
@media (prefers-reduced-motion: no-preference) {
  .live-dot { animation: pulse-live 2s ease-in-out infinite; }
  .sheet-open { animation: slide-up 0.25s ease-out; }
}
```

## 8. Performance Rules
 
These are mandatory — the app will be used on Jio 4G in rural Kerala:
 
- **No image files in v0** — use emoji and CSS only
- **No Google Fonts import** — use system font stack (`Inter` loads from system on most Android devices; fallback to `system-ui`)
- Bundle size target: **< 150KB JS** for first load
- All Supabase queries must have a **5-second timeout** with graceful error state
- Gate list must render within **1 second** on slow 4G (use Suspense + skeleton loaders)
- Skeleton loader for gate cards: `--bg-elevated` placeholder with shimmer animation
- Mapbox GL: **lazy load only on map tab** — do not import on the list page

## 10. Map Page (V2) Rules
 
- Default map style: `mapbox://styles/mapbox/dark-v11` — matches app theme
- Gate pins: custom markers (not default Mapbox teardrops)
  - Open gate:   green circle 16px with white border
  - Closed gate: red circle 16px with white border
  - Unknown:     gray circle 12px
- On pin tap: show a compact popup (not a full modal) with gate name + status + "Report" button
- Map controls: only zoom in/out — hide compass and other defaults
- Attribution: keep Mapbox attribution (required by license) but move to bottom-left
- User location dot: blue pulsing circle — only show if user grants location permission
- Never auto-fly to location without user action

## 11. Copy & Language Rules
 
- App name: always lowercase — **railundo** (not RailUndo in UI text, only in manifest/meta)
- Question format: **"Gate undo?"** — follows Undo Universe convention
- Status labels: OPEN / CLOSED / UNKNOWN — always uppercase
- Timestamps: relative — "2 min ago", "just now", "12 min ago" — never absolute time
- Report count: "3 reports" not "3 users reported"
- Error messages: plain Malayalam-English — "Something went wrong. Try again." — never technical errors to user
- Empty state: conversational — "No reports yet. Be the first!" 
- CTA button: "Report Status" — not "Submit" or "Update"

## 12. File & Code Conventions
 
```
All Tailwind classes must use CSS variables where possible via arbitrary values:
  bg-[var(--bg-surface)]  text-[var(--text-primary)]
 
Define all CSS variables in globals.css :root { } — not inline
 
Component file naming: PascalCase — GateCard.tsx, DistrictFilter.tsx
Utility file naming: camelCase — formatRelativeTime.ts, getStatusColor.ts
 
All components must be mobile-first — no desktop-first overrides
Breakpoints: only use sm: (640px) and above for larger screen centering
Default styles = mobile. sm: styles = desktop centering only.
 
Tailwind config must extend theme with the color variables above.
```

## 13. What to Never Build
 
- No login / signup flow — ever in v0 or v1
- No user profiles
- No comments or social features
- No ads or promotional banners
- No splash screen / loading screen longer than 300ms
- No horizontal carousels on the main list
- No modals that cover the full screen (use bottom sheets instead)
- No pagination — use infinite scroll or "Load more" for gate list
- No tooltips — if something needs a tooltip, redesign it
